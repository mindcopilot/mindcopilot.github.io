写完上一篇《学习 AI Agent》没几天，我就遇到了一个很好的「实物教材」：**Hermes Agent**。

如果说上一篇讲的是 Agent 的「原理」，那这一篇就是把原理对照到一个真实项目上来看。而 Hermes Agent 最打动我的地方，恰恰是我上一篇里特意点名「最容易被低估」的那个概念 —— **记忆**。

## Hermes Agent 是什么

Hermes Agent 是 **Nous Research** 在 2026 年 2 月开源的一个自主智能体项目，代码在 GitHub 上以 MIT 协议放出。它的一句话定位很直白：

> 「一个会和你一起成长的智能体（the agent that grows with you）。」

和我们平时用的对话式 AI 不同，Hermes Agent 被设计成**长期在线**的：它跑在你自己的一台服务器上（据介绍，一台很便宜的 VPS 就够用），持续运行、持续积累。你用自己的 API Key 接入模型 —— 它支持 OpenAI、Anthropic，也支持通过 Ollama 跑本地模型 —— 数据留在你自己机器上。

发布之后它在开源社区涨得非常快，短时间内就积累了数以万计的 GitHub Star。但比「火」更值得说的，是它**为什么**这样设计。

## 它要解决的问题：AI 的「失忆」

你大概也有过这种体验：和一个 AI 助手聊得很顺，把背景、偏好、上下文都交代清楚了 —— 然后关掉窗口，下次再打开，它什么都不记得，你得从头再来一遍。

大多数智能体也是这样：**每个任务都从一张白纸开始。** 它们会推理、会调工具，但不会「积累」。

Hermes Agent 的核心主张就是冲着这件事来的：**一个真正有用的智能体，应该会记住。** 它把「记忆」从一个附属功能，提到了整个设计的中心。

## 三个关键设计

我把它的设计拆成三块来理解。

### 一、分层的持久记忆

Hermes Agent 不是把所有对话一股脑塞进上下文，而是用了一套**分层记忆**：

- 据资料介绍，它用 **SQLite 的全文检索（FTS5）** 加上**大模型摘要**来管理记忆 —— 重要的洞察被提炼进「内部记忆」，其余的留在「资料库」里待查。
- 记忆不是一个越长越臃肿的大文件。它采用**索引式结构**：把一个总的 `MEMORY.md` 拆成一个索引文件加多个子文档，**运行时按需加载**。理解起来大致是这个意思：

```
MEMORY.md            索引：登记有哪些子文档、各自讲什么
memory/
  preferences.md     你的偏好与习惯
  project-notes.md   某个项目的细节
  ......              用到哪个，才加载哪个
```

这个设计很聪明：它既让智能体「记得多」，又不会让每次调用都背着一大坨记忆，把上下文窗口和成本撑爆。这也正好回应了我上一篇里那句话 —— 「记住该记的、忘掉该忘的，本身就是一门手艺」。

### 二、技能的自我沉淀

第二块是我觉得最有意思的：**学习循环**。

普通智能体每次都从零解题。Hermes Agent 则会把一次**成功解决的流程，沉淀成一个可复用的「技能」**：

- 技能以可检索的 Markdown 文件形式保存，遵循 `agentskills.io` 这个开放标准；
- 下次遇到类似任务，它可以加载旧技能、套用已有步骤；
- 而且技能会**越用越好** —— 如果某个技能在第二次使用时漏掉了边界情况，智能体会回头把它补上。

据 Nous Research 自己的基准测试，一个攒了 20 个以上自创技能的智能体，完成研究类任务比「全新实例」快了约 40%，而且不需要人去手动调提示词。

换句话说，它不只是「会做事」，还会**把做过的事变成自己的能力**。

### 三、长期在线，多个入口

第三块偏工程，但很实在。Hermes Agent 常驻在服务器上，通过一个统一网关接入 Telegram、Slack、Discord、WhatsApp 等聊天平台，以及命令行；它还能跑定时自动化任务，内置了几十个工具。

也就是说，它不是你「打开来用一下」的 App，而是一个**一直在那儿、随时能找你、也随时能被你找到**的存在。

## 我的几点思考

把这三块拼起来，Hermes Agent 给我的感觉是：它想把智能体，从一个**一次性的工具**，变成一个**长期的伙伴**。

这里面有让我兴奋的地方：

- **记忆改变了关系。** 一个记得住你的智能体，和一个每次失忆的，完全是两种东西 —— 前者才谈得上「越用越合手」。
- **自托管加数据本地**，意味着这种「长期关系」建立在你自己掌控的地基上，而不是租来的。

但也有让我保持警惕的地方：

- 一个会**自我进化**的智能体，在积累「能力」的同时，也会积累**错误和偏见**。技能库里要是沉淀进了一个有问题的做法，它可能会一错再错。
- 「记住」是把双刃剑 —— 它记住的东西，也需要有人去**审视和管理**。记忆越长，越需要知道里面装了什么。

说到底，这又回到了我一直念叨的那句话：工具越强、越主动，**怎么用好它、怎么为它的「成长」把关，就越是人的责任。** 一个会成长的智能体，更需要一个会判断的人。

## 小结

三句话留给自己：

- Hermes Agent 是一个以**记忆**为中心的开源自主智能体，目标是治好 AI 的「失忆」。
- 三个关键设计：分层持久记忆、技能的自我沉淀（学习循环）、长期在线的多入口运行。
- 它把智能体从「一次性工具」推向「长期伙伴」—— 而伙伴关系，意味着信任，也意味着责任。
